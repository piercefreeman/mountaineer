## First-Run Commands
1. Start Postgres: `docker compose up -d postgres`
2. Initialize schema (preferred):
{% if package_manager == 'poetry' %}
   - `poetry run migrate generate --message init`
   - `poetry run migrate apply`
{% elif package_manager == 'uv' %}
   - `uv run migrate generate --message init`
   - `uv run migrate apply`
{% else %}
   - `source venv/bin/activate`
   - `migrate generate --message init`
   - `migrate apply`
{% endif %}
3. Start server:
{% if package_manager == 'poetry' %}
   - `poetry run runserver`
{% elif package_manager == 'uv' %}
   - `uv run runserver`
{% else %}
   - `runserver`
{% endif %}

`createdb` is available as a one-shot bootstrap fallback.

## Engineering Rules
- Keep changes small and task-focused.
- Prefer `rg` for search and avoid destructive git commands.
- Preserve typing and existing architecture patterns.
- If table models change, create/apply migrations as part of the workflow.
- Validate touched areas before finishing.
